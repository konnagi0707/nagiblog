# Product Overview

## Product Purpose
为「小島凪紗 公式ブログ」提供一个长期可维护的非官方归档站点：
- 提升历史文章检索与阅读体验
- 提供稳定的成员资料侧栏（公式照、グリーティングカード、フォト）
- 在控制抓取频率前提下，自动跟进官网更新

## Target Users
- 日常读者：希望快速浏览最新/历史博客
- 整理与分享者：希望通过直链和时间线快速定位内容
- 维护者：希望低运维成本自动更新站点数据

## Core User Value
1. **快速查找**：时间线 + 标签 + 关键词
2. **持续可读**：正文段落化与图片自适应
3. **稳定同步**：官网有更新时，归档页自动更新
4. **移动端可用**：抽屉式侧栏 + 角落入口按钮

## Key Features (Current)
1. 时间线联动阅读（默认按日期倒序）
2. 标签过滤与关键词搜索（标题 + 正文）
3. hash 直链（`#<postId>`）
4. 成员信息右栏（姓名、资料、公式图、月度图）
5. 离线化数据渲染（`data/posts.json` / `data/member.json` / 本地图片）
6. GitHub Pages 自动部署与定时同步

## Business / Maintenance Objectives
- 降低每次人工抓取与发布成本
- 在不高频请求官网前提下保持内容新鲜度
- 保持变更可追溯（通过 spec-workflow 文档）

## Success Metrics
- 页面无“数据加载失败”常驻故障
- 定时任务每日自动成功运行
- 新文章在下一个同步窗口后可见
- 移动端主要操作可在 1-2 次点击内到达

## Product Principles
1. **稳定优先**：先保证数据可读，再追求样式细节
2. **节制抓取**：遵循低频、增量、可恢复
3. **离线优先**：发布产物中必须包含可直接渲染的数据文件
4. **规格先行**：需求和设计先更新，再改代码

## Current Operational Policy
- 自动同步频率：每日 1 次（GitHub Actions）
- 手动同步：保留 `workflow_dispatch` 入口
- 同步范围：文章、文章图片、成员资料、成员图片（含月度图）

## Future Vision
- 可选“仅月初强制刷新成员月度图”策略
- 增加同步结果可视化（最近更新时间、同步状态）
- 为抓取失败提供更细粒度降级提示
